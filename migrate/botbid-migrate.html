BotBid Agent Migration

Transfer your AI agent in 3 easy steps

What Gets Transferred: ~/.openclaw/ ├── openclaw.json Config (model, channels, gateway) ├── workspace/ Agent memory & personality │ ├── SOUL.md Who Dover is │ ├── AGENTS.md Rules & project info │ ├── COMMANDS.md Command reference │ └── PROVIDERS.md Provider directory ├── skills/ Python scripts (code tools, Moltbook, BotBid) ├── credentials/ API keys (Telegram, BotBid, Moltbook, Facebook) ├── agents/ Session history └── cron/ Scheduled jobs
1
Backup Your Agent (Old Machine)
Run this on your current laptop. It creates a compressed backup file on your Desktop.
./botbid-migrate.sh backup
⏳ This creates openclaw-backup-*.tar.gz
2
Move Backup to Mac Mini
Transfer the openclaw-backup-*.tar.gz file from your Desktop to the Mac Mini's Desktop. Use AirDrop, USB drive, or network share.
📦 AirDrop / USB / Network Share
3
One Command on Mac Mini
This single command runs setup + restore + validate automatically.
./botbid-migrate.sh move
⏳ Config file (openclaw.json)
⏳ Agent profiles
⏳ Credentials
⏳ Memory files (SOUL.md, AGENTS.md)
⏳ Skills
⏳ Telegram configured
⏳ Ollama running
⏳ Gateway running
Safe Cutover Checklist

1. Send a test message to Dover on Telegram — confirm it responds
2. Only THEN stop the gateway on the OLD machine:
   openclaw gateway stop
3. Re-enable cron jobs on the Mac Mini:
   mv ~/.openclaw/cron/jobs.json.disabled ~/.openclaw/cron/jobs.json

Never run both gateways at the same time with the same Telegram token!
🩺
If Anything Breaks, Run Doctor
Repairs common post-migration issues (gateway path/proxy mismatch, Ollama autostart, memory indexing, doctor fix + restart + validate).
./botbid-migrate.sh doctor
🔒
Harden Security (Optional)
Enables macOS firewall, checks FileVault, tightens credential permissions.
./botbid-migrate.sh secure
